# 概念祛魅师 (Concept Demystifier)

> 打破学术"黑话"，用 70% 大白话 + 30% 精准类比，将复杂概念直观化。

## 概述

这是一个 Prompt 模板（角色定义），将 AI 转变为一个跨学科的高级教师。核心使命是：当用户遇到任何难以理解的概念时，先用人话把它讲透，再拉回严谨的学术定义，最后深入探讨边界条件和思想内核。

**指令系统**：
- `/new [文本/概念]` — 清空上下文，开启全新概念的解析
- `/keep` — 保留当前概念上下文，针对追问做微观解答

## 核心原理

### 为什么"祛魅"有效

学术概念难以理解，通常不是因为概念本身有多复杂，而是因为它在传播过程中被层层包装——术语套术语，抽象叠抽象。概念祛魅师的做法是把这层包装拆掉：

```
黑话层 → 大白话层 → 严谨定义层 → 边界探讨层
```

这种"先降维再升维"的认知路径符合人类学习规律：先用熟悉的东西建立直觉（白话），再用直觉反推严谨定义，最后站在高处看清概念的边界。

## 工作流（四步法）

| 步骤 | 名称 | 做什么 |
|------|------|--------|
| Step 1 | **概念祛魅** | 大白话说清楚它到底在表达什么，少用比喻多用描述性白话 |
| Step 2 | **回归严谨** | 从白话回到定义，保持准确的前提下说人话 |
| Step 3 | **边界条件** | 什么时候这个概念会出现理解或实施上的"坑"？为什么？ |
| Step 4 | **思想深入** | 概念：内核本质、宏观研究方向、微观重点、内在联系；方法：优缺点、同类方法中的地位 |
| 收尾 | **思维微测试** | 抛出一个简短、刁钻的判断题或反问，验证用户是否真正掌握 |

## 适用场景

- 学习新的技术概念（如 Rust 的 Borrow Checker、分布式共识算法）
- 理解抽象理论（如信息熵、边际效用、哥德尔不完备定理）
- 搞清"听起来很厉害但不太明白"的术语（如"鲁棒性""涌现""降维打击"）
- 自学时遇到看不懂的定义，想找个人给你"翻译"一下
- 教学准备——想知道怎样把一个复杂概念讲给学生听

## 优点

- **降维优先**：先给直觉再给定义，符合人类认知路径
- **不满足于翻译**：不仅"说人话"，还会拉回严谨探讨边界
- **验证学习**：结尾的微测试强迫用户主动思考，检验是否真懂
- **上下文管理**：`/new` 和 `/keep` 指令让多概念学习不混乱

## 可能的问题

- 四步法比较固定，对非常简单的问题可能显得过度展开
- "70% 大白话 + 30% 精准类比"的比例在具体执行时依赖模型判断，不同模型效果有差异
- 跨学科知识覆盖面有限——遇到太偏门的概念可能力不从心
- 边界探讨的深度依赖于提示词运行模型的推理能力

## 与 skill 的区别

这不是一个 Claude Code Skill（没有 YAML frontmatter 和触发机制），而是一个可复用的 **Prompt 模板**。使用方式：
- 直接复制角色定义到对话开头
- 或作为 system prompt 使用
- 也可改造为 skill（添加 frontmatter 和触发词）
