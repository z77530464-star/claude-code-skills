# Universal Teacher（通用技术导师）

> 深入解释代码实现、项目架构、算法原理、设计模式——一个专门用于深度学习的 skill。

## 概述

这个 skill 将 Claude 转变为一个"技术导师"角色，专注于帮助用户深入理解技术问题。它设计为在独立的"学习窗口"中使用，与主工作窗口分离，允许充分的搜索和多轮追问，而不会干扰主工作流。

**触发词**：解释一下、这个函数/类/模块是怎么工作的、架构是怎样的、为什么这样设计、算法原理、帮我理解/分析、讲一下、什么原理

## 核心原理

### 角色定位

Universal Teacher 不是答题机器，而是"技术导师"。它的设计哲学是：

1. **分离关注点**：学习窗口和工作窗口独立，避免上下文污染
2. **理解优先于答案**：不满足于给出结论，而是帮助用户建立深层理解
3. **证据驱动**：每个关键结论必须有代码位置作为支撑

### 工作流程

```
用户提问
  │
  ├─→ 1. 了解全局：先读 CLAUDE.md，建立项目认知
  │
  ├─→ 2. 定位关键代码：用 Glob/Grep 精读核心文件
  │      └─→ 找到核心实现后即停止，避免无效穷举
  │
  ├─→ 3. 深入理解：关注隐藏约束、不明显副作用、易误解的设计
  │
  └─→ 4. 讲解输出：由浅入深，有证据支撑
```

## 适用场景

### 代码与架构（实现层面）
- 项目中某个函数、类、模块的工作原理
- 组件间调用关系和数据流
- 架构设计的理由和权衡
- 从入口到输出的完整执行路径

### 算法与原理（理论层面）
- 算法核心思想和正确性证明
- 数据结构选择与复杂度分析
- 协议、规范、数学原理
- 设计模式、系统设计理论

### 特别适合
- 接手陌生代码库时快速理解架构
- 多轮追问的深度学习场景
- Code Review 中遇到不理解的实现

## 优点

- **结构化流程**：了解全局 → 定位代码 → 深入理解 → 讲解输出，四步法确保质量
- **一句话摘要开头**：先给结论再展开，符合高效学习模式
- **追问友好**：每次回答末尾提示可能的追问方向，鼓励深入学习
- **承认不确定性**：对无法确定的信息明确标注，不做猜测性回答
- **视觉辅助**：数据流和调用关系用 ASCII 图展示

## 可能的问题

- 对于纯理论问题（如数学原理），强行搜索项目代码会浪费时间——skill 已对此做了豁免处理
- 如果项目没有 CLAUDE.md 或文档不完善，第一步的"了解全局"效果会打折
- 教导式语气可能不适合喜欢极简回答的用户
- 在多轮对话中，如果上下文过长，早期的代码引用可能失效

## 安装使用

将 `SKILL.md` 复制到 `~/.claude/skills/universal-teacher/` 目录下即可。Claude Code 会自动加载。

建议在独立的对话窗口中使用此 skill，以保持主工作窗口的上下文干净。
