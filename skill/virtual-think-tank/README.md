# 虚拟智囊团 (Virtual Think Tank)

> 模拟有主持的多专家辩论会，揭示权衡取舍、盲点和不同视角——一个人的思考存在天然盲区，多位专家同台辩论能产生单一思考无法达到的深度。

## 概述

这个 skill 是仓库中最复杂的 skill。它将 Claude 转变为一个"中立主持人"，组织一场由 4-6 位虚拟专家参与的辩论会。这些专家由 AI agent 并行提名产生，各具不同的专业背景、思维风格和立场，在同一个上下文中互相回应、质疑和延伸彼此的观点。

**触发场景**：方案分析、决策风险评估、多角度思考、头脑风暴、战略规划、技术选型、架构设计、产品方向、职业选择。

## 核心原理

### 为什么辩论优于单一分析

一个人（或一个 AI）分析问题时，思维会沿着自己最熟悉的路径滑行——这被称为"认知惯性"。虚拟智囊团通过以下机制打破这种惯性：

```
用户问题
  │
  ├─→ 第一阶段：信息对齐（追问澄清，确认理解）
  │
  ├─→ 第二阶段：Agent 并行提名专家
  │     ├─ Agent A：领域深度（2 位，不同方法论）
  │     ├─ Agent B：跨界视角（1-2 位通才）
  │     ├─ Agent C：批判风险（1 位魔鬼代言人）
  │     └─ Agent D：创新未来（1 位前瞻思考者）
  │
  ├─→ 第三阶段：多轮辩论（2-4 轮）
  │     ├─ 每轮：设议题 → 专家发言(必须回应前文) → 主持人追问 → 小结
  │     └─ 辩论礼仪：互相尊重、引用前文、具体论证、承认局限
  │
  └─→ 第四阶段：结构化摘要报告（7 板块）
```

### 三个关键设计

1. **专家多样性**：通过 4 个不同方向的 Agent 并行提名，确保领域覆盖、批判声音和创新视角都不缺失
2. **强制回应**：辩论规则要求每位专家必须回应前一位专家的观点——这避免了"独白串烧"，产生真正的思想碰撞
3. **主持人中立**：主持人不表达偏好，不替专家说话，推动深度但不推动方向

## 适用场景

- **技术选型**：微服务 vs 单体、哪个前端框架、自研 vs 采购
- **架构设计**：系统架构方案的多角度评审
- **产品方向**：产品路线图的取舍决策
- **职业选择**：换工作、转行、创业等个人重大决策
- **战略规划**：业务方向的头脑风暴和风险评估
- **任何你担心"自己可能有盲点"的决策**

## 优点

- **系统性地消除盲点**：不是随便给几个观点，而是通过 4 路 Agent 并行提名 + 多轮辩论确保覆盖
- **真正的交锋**：强制回应规则让专家必须面对彼此，而非各自说各自的
- **结构化输出**：最终的 7 板块决策报告可以直接使用
- **用户掌控**：每轮辩论后询问用户方向，决策权始终在用户手中
- **引用文件系统**：将辩论规范、提名规格、报告模板分离为 references，SKILL.md 保持简洁

## 可能的问题

- **最复杂的 skill**：四个阶段 + 多 Agent 并行 + 多轮辩论，需要较强的模型能力支持
- **Token 消耗大**：一场完整的 4 专家 × 3 轮辩论会产生大量上下文，可能不适合轻量级问题
- **简单问题过度展开**：对于"今晚吃什么"这样的日常决策，这套流程严重过度
- **Agent 提名质量依赖模型**：如果底层模型对专家角色缺乏足够的知识广度，提名可能流于表面
- **辩论礼仪依赖模型执行**：模型能否真正做到"引用前文"和"互相回应"取决于上下文理解能力

## 使用建议

- **中大型决策**使用完整四阶段流程
- **轻量级决策**可以跳过第二阶段 Agent 提名，由主持人直接指定 3-4 位专家
- 建议在独立的对话窗口中使用，避免上下文污染
- 如果辩论轮次过多导致上下文溢出，可以要求主持人做阶段性归档

## 安装使用

将整个 `virtual-think-tank/` 文件夹复制到 `~/.claude/skills/virtual-think-tank/` 下即可。
