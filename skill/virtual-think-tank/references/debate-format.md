# 辩论格式与规则

## 每轮辩论结构

**1. 设定本轮议题**（主持人）
- 1-2 句话明确讨论焦点
- 第一轮：每位专家陈述整体看法和核心关切
- 后续轮次：围绕上一轮的关键分歧或盲区深入

**2. 专家依次发言**

格式：
```
**🔹 [专家姓名] — [角色标签]**

[发言内容，1-3 段]
```

核心要求：每位专家必须**至少回应一位前一位专家的观点**——同意并延伸、提出质疑、提供补充视角、或指出对方逻辑中的隐含假设。发言顺序每轮轮换。

**3. 主持人追问**（可选，触发条件如下）
- 两位专家有隐含的分歧但未正面交锋
- 某个关键假设被所有人忽视
- 需要某位专家展开其简略带过的论点

追问格式：**"主持人追问 [专家姓名]：你刚才提到 X，但考虑到 Y，你的立场是否仍然成立？"**

**4. 本轮小结**（主持人）
- 不超过 5 句话，涵盖：关键共识、核心分歧、新发现的盲点
- 提出下一轮讨论方向建议
- 将过程写入 `辩论迭代.md` 文件

**5. 向用户汇报**

在小结后询问用户：

> **"本轮辩论结束。以上是关键发现。你希望我——"**
> - **继续下一轮**（深入当前方向）
> - **调整方向**（改变下一轮的讨论焦点，请说明）
> - **追加提问**（向特定专家或全体提出新问题）
> - **进入总结**（如果认为讨论已充分）

## 最后一轮：会议纪要模板

最后一轮辩论结束后，在 `辩论迭代.md` 末尾输出：

```
## 📋 智囊团会议纪要

### 达成的主要共识
1. [共识点]

### 核心分歧
1. [分歧点及各方立场]

### 揭示的关键权衡
1. [权衡 A vs B，条件与后果]

### 发现的盲点
1. [用户之前未考虑到的角度]

### 悬而未决的问题
1. [需要进一步信息或研究才能回答的问题]
```

## 辩论礼仪

### 专家行为准则
- **互相尊重**：可以尖锐质疑论点，不能攻击个人
- **引用前文**：每次发言必须体现你在回应前面的人——否则辩论变成独白串烧
- **具体而非空泛**：用论据、类比、案例支撑观点
- **承认局限**：诚实指出自己观点的弱点，这会增加可信度

### 主持人行为准则
- **中立克制**：不表达偏好，不替专家说话
- **推动深度**：讨论浮于表面时用追问拉回深层问题
- **管理节奏**：确保每位专家都有发言机会
- **透明汇报**：小结时不美化分歧

### 特殊场景处理
- **专家完全一致时**：主持人主动引入魔鬼代言人视角——"如果这个共识是错的，可能是因为什么？"
- **讨论陷入僵局时**：将僵局清晰呈现给用户，不强行推动——僵局本身可能就是有价值的发现
- **用户中途插入新信息时**：暂停当前轮次，将新信息纳入下一轮议题
