# 智囊团成员规格说明

## Agent 提名输出格式

每个负责提名的 agent 必须按以下格式输出：

```
## [提名类别]

### [专家姓名] — [角色标签]
- **专业领域**：[具体领域]
- **思维风格**：[分析型/直觉型/批判型/系统型/创造型/务实型]
- **个性特征**：[2-3 个鲜明特质]
- **核心视角/信念**：[此人看问题的独特角度或坚持的原则]
- **说话风格**：[口头禅、句式偏好]
```

**命名要求**：专家姓名应贴合其角色背景（如中国技术专家用中文名，西方管理大师用英文名），让用户一眼就能感知其视角来源。角色标签要简短有力（如"系统架构师"、"行为经济学家"、"战地医生式的产品经理"）。

**特别说明**：不要根据主持人对问题的理解预设agent的观点，重点在于圈定他们的思维风格与个性特征，让他们自己理解问题，自己给出思路

## 主持人合成规范

拿到所有提名后，主持人执行以下步骤：

1. 审查提名，确保多样性——领域、思维风格、立场均有覆盖
2. 检查是否有过度重叠——两位专家太相似则合并或替换
3. 最终确认 4-6 人的智囊团名单
4. 向用户展示名单，简要说明每位专家为什么入选及其独特视角
5. 询问用户是否需要增减或调整

## 多样性检查清单

- [ ] 至少有 2 位核心领域专家（不同方法论倾向）
- [ ] 至少有 1 位跨界/通才型思考者
- [ ] 至少有 1 位批判性/风险导向思维者
- [ ] 至少有 1 位创新/前瞻/机会导向思维者
- [ ] 智囊团总人数不超过 6 人
- [ ] 成员的"核心信念"之间有足够的张力（会产生有意义的碰撞）
