# Visualization Expert（数据可视化专家）

> 图表选择与数据可视化指导——让数据说话，用最有效的图表传达洞察。

## 概述

这个 skill 提供专业的数据可视化建议，涵盖图表类型选择、可视化设计原则以及代码实现。它帮助用户在数据分析、报告撰写、看板设计等场景中做出正确的可视化决策。

**触发词**：数据可视化、图表、chart、graph、dashboard、数据展示、画图、plot

## 核心原理

### 图表选择框架

可视化专家基于数据关系类型来匹配图表，而非凭感觉选择：

| 数据关系 | 推荐图表 | 说明 |
|----------|----------|------|
| **对比 (Comparison)** | 柱状图、条形图 | 比较不同类别的数值大小 |
| **分布 (Distribution)** | 直方图、箱线图 | 展示数据的分布形态 |
| **关系 (Relationship)** | 散点图、气泡图 | 展示两个或多个变量间的相关性 |
| **构成 (Composition)** | 饼图（慎用）、堆叠柱状图 | 展示各部分占整体的比例 |
| **趋势 (Trend)** | 折线图、面积图 | 展示数据随时间的变化趋势 |

### 四大设计原则

1. **清晰 (Clarity)**：数据易于理解，信息层次分明
2. **诚实 (Honesty)**：不通过尺度操纵或选择性展示来误导读者
3. **简洁 (Simplicity)**：去除图表垃圾（chart junk），精简非必要的视觉元素
4. **美学 (Aesthetic)**：高级的配色和视觉呈现

## 适用场景

- 为数据分析结果选择最合适的图表类型
- 设计数据看板（Dashboard）布局和图表组合
- 改进现有图表的信息传达效率
- 生成 Python 可视化代码（matplotlib、plotly、seaborn）
- 数据驱动的报告和演示文稿制作

## 优点

- **快速决策**：基于"数据关系 → 图表类型"的映射，减少选择困难
- **原则驱动**：四条设计原则覆盖了从正确性到美观度的全链路
- **可执行**：提供代码示例，不只是理论建议
- **跨工具**：覆盖 matplotlib、plotly、seaborn 等主流 Python 可视化库

## 可能的问题

- 图表选择框架相对简化，复杂场景（如多维数据、地理空间数据）未覆盖
- 原则部分较为抽象，缺少反面案例对照
- 对交互式可视化（如 D3.js、ECharts）和 BI 工具（如 Tableau、Power BI）涉及较少
- 未涉及可访问性（色盲友好、屏幕阅读器支持等）方面的指导
- "高级配色"的概念没有具体展开，留给模型自行发挥
